# Specs: User Stories

## Персоны
- **Пользователь:** ведет личные заметки на одном компьютере, ожидает быстрый поиск и надежное сохранение.

## User Stories

### US-1: Создание заметки
**Как** пользователь, **хочу** создавать заметки с заголовком и текстом, **чтобы** сохранять идеи и задачи.
- **Критерии приемки:**
  - Можно создать заметку без тегов.
  - Заголовок обязателен (не пустой после trim), текст может быть пустым.
  - После создания заметка появляется в списке и доступна для просмотра/редактирования.
  - После перезапуска приложения заметка сохраняется.

### US-2: Редактирование заметки
**Как** пользователь, **хочу** редактировать заголовок и текст, **чтобы** актуализировать информацию.
- **Критерии приемки:**
  - Изменения сохраняются при явном действии “Сохранить” или при переключении на другую заметку.
  - При ошибке записи показывается сообщение и изменения не теряются (остаются в UI до успешного сохранения/отмены).

### US-3: Удаление заметки
**Как** пользователь, **хочу** удалять заметки, **чтобы** очищать список.
- **Критерии приемки:**
  - Перед удалением есть подтверждение.
  - Удаленная заметка не появляется в списке и не возвращается после перезапуска.

### US-4: Быстрый поиск
**Как** пользователь, **хочу** искать заметки по подстроке, **чтобы** быстро находить нужную.
- **Критерии приемки:**
  - Поиск работает по заголовку и тексту.
  - Результаты обновляются при вводе.
  - Поиск регистронезависимый.

### US-5: Теги и фильтрация
**Как** пользователь, **хочу** назначать заметкам теги и фильтровать по ним, **чтобы** группировать заметки.
- **Критерии приемки:**
  - Тег — это короткая строка (1–32 символа), без пробелов по краям.
  - Заметке можно назначить несколько тегов.
  - Фильтр по тегу показывает только заметки с выбранным тегом.

### US-6: Закрепление (Pin)
**Как** пользователь, **хочу** закреплять важные заметки, **чтобы** они всегда были сверху.
- **Критерии приемки:**
  - Закрепленные заметки отображаются первыми в списке.
  - При равных условиях сортировки более новые заметки выше.

### US-7: Экспорт/импорт
**Как** пользователь, **хочу** экспортировать и импортировать заметки, **чтобы** делать бэкап или перенос.
- **Критерии приемки:**
  - Экспорт создает JSON файл со всеми заметками и тегами.
  - Импорт добавляет новые заметки, не дублируя существующие по (title, created_at) в рамках примера.
  - При конфликте показывается итоговая статистика: добавлено/пропущено/ошибок.
