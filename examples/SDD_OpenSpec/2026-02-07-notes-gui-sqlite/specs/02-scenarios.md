# Specs: Scenarios, Invariants, Rules

## Сценарии (Given/When/Then)

### SC-1: Создание заметки и сохранение
- **Дано:** приложение запущено, база доступна
- **Когда:** пользователь вводит заголовок “Идея” и текст “Сделать прототип” и нажимает “Создать”
- **Тогда:** заметка появляется в списке, в базе появляется запись, после перезапуска заметка остается

### SC-2: Редактирование с ошибкой записи
- **Дано:** открыта существующая заметка, база становится недоступной (например, файл заблокирован)
- **Когда:** пользователь меняет текст и нажимает “Сохранить”
- **Тогда:** показывается сообщение об ошибке, UI сохраняет несохраненные изменения и позволяет повторить попытку или отменить

### SC-3: Поиск по подстроке
- **Дано:** есть заметки “Покупки” и “Покупки на выходные”
- **Когда:** пользователь вводит в поиск “пок”
- **Тогда:** обе заметки отображаются в результате, регистр не влияет

### SC-4: Фильтр по тегу
- **Дано:** заметка A с тегом “work”, заметка B с тегом “home”
- **Когда:** пользователь выбирает фильтр “work”
- **Тогда:** в списке отображается только заметка A

### SC-5: Закрепление и сортировка
- **Дано:** две заметки, одна закреплена
- **Когда:** пользователь открывает список
- **Тогда:** закрепленная заметка выше, независимо от даты остальных

### SC-6: Импорт без дублей в учебном режиме
- **Дано:** в базе есть заметка с title “Идея” и created_at = T1
- **Когда:** пользователь импортирует JSON, где есть заметка с тем же title и created_at = T1
- **Тогда:** заметка считается дублем и пропускается, это отражено в статистике

## Инварианты
- Каждая заметка имеет уникальный идентификатор (id) в базе.
- Каждая заметка имеет непустой title после trim.
- Время created_at неизменно после создания, updated_at обновляется при успешном сохранении.
- Теги в рамках приложения уникальны по нормализованному имени (trim, lower).

## Бизнес-правила
- В UI запрещено создавать/сохранять заметку с пустым заголовком.
- Поиск регистронезависимый, по подстроке, работает по title и body.
- Удаление физическое (для простоты примера), без корзины.

## Формат экспорта (учебный)
- Экспортируемый JSON содержит:
  - `notes[]`: `{ title, body, is_pinned, created_at, updated_at, tags[] }`
- Поля `id` и внутренние версии схемы не экспортируются, чтобы показать необходимость дедупликации при импорте.
