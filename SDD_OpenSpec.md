# OpenSpec SDD Template
*Основано на методологии [Fission-AI OpenSpec](https://github.com/fission-ai/openspec)*

## Основные артефакты OpenSpec

Процесс разработки OpenSpec опирается на четыре ключевых документа (артефакта), которые создаются для каждого изменения:

1.  **proposal.md (Предложение)**
    *   *Назначение:* Описывает **ЗАЧЕМ** мы это делаем и **ЧТО** именно меняется.
    *   *Когда:* В самом начале, для инициации и обоснования изменения.
    *   *Содержание:* Обоснование, область действия, критерии успеха, не-цели.
    *   *Аналог:* Business Requirements Document (BRD) / Project Charter.

2.  **specs/ (Спецификации)**
    *   *Назначение:* Описывает **ТРЕБОВАНИЯ** и **СЦЕНАРИИ** использования.
    *   *Когда:* После утверждения идеи (Proposal), но перед техническим проектированием.
    *   *Содержание:* Пользовательские истории, сценарии (Given-When-Then), инварианты, бизнес-правила.
    *   *Аналог:* Functional Requirements Specification (FRS) / User Stories.

3.  **design.md (Технический дизайн)**
    *   *Назначение:* Описывает **КАК** это будет реализовано технически.
    *   *Когда:* После согласования требований (Specs), перед написанием кода.
    *   *Содержание:* Архитектура, модели данных, API, потоки данных, риски, план миграции.
    *   *Аналог:* Software Design Document (SDD) / Technical Design Document (TDD).

4.  **tasks.md (Задачи)**
    *   *Назначение:* Пошаговый **ПЛАН** реализации.
    *   *Когда:* Непосредственно перед кодированием, для декомпозиции работы.
    *   *Содержание:* Чек-лист конкретных действий (Setup, Backend, Frontend, Testing), разбитый на фазы.
    *   *Аналог:* Work Breakdown Structure (WBS) / Backlog.

## 0. Onboarding & Prerequisites
**Цель:** Быстрое подключение к OpenSpec и подготовка окружения.

### Требования и установка
*   **Node.js:** ≥ 20.19.0
*   **Установка:** `npm install -g @fission-ai/openspec@latest`
*   **Инициализация:** `openspec init`
*   **Обновление инструкций агента:** `openspec update`

### Рекомендации
*   **Модели:** высоко-логические (например, Opus 4.5, GPT 5.2)
*   **Гигиена контекста:** очищайте контекст перед реализацией
*   **Телеметрия (опционально):** `OPENSPEC_TELEMETRY=0` или `DO_NOT_TRACK=1`

## 5. Workflow (Команды и артефакты)
**Цель:** Согласованный рабочий процесс OpenSpec.

### Команды
*   `/opsx:onboard` — подготовка проекта и агента
*   `/opsx:new <slug>` — завести изменение и структуру артефактов
*   `/opsx:ff` — сгенерировать proposal, specs, design, tasks
*   `/opsx:apply` — выполнить задачи
*   `/opsx:archive` — архивировать изменение и обновить общие specs

### Артефакты изменения
```
openspec/
  changes/
    <YYYY-MM-DD>-<slug>/
      proposal.md
      specs/
      design.md
      tasks.md
```
